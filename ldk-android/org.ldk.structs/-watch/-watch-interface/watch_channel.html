abstract fun watch_channel(funding_txo: OutPoint, monitor: ChannelMonitor): Result_NoneChannelMonitorUpdateErrZ

Watches a channel identified by `funding_txo` using `monitor`. Implementations are responsible for watching the chain for the funding transaction along with any spends of outputs returned by [`get_outputs_to_watch`]. In practice, this means calling [`block_connected`] and [`block_disconnected`] on the monitor. Note: this interface MUST error with `ChannelMonitorUpdateErr::PermanentFailure` if the given `funding_txo` has previously been registered via `watch_channel`.
